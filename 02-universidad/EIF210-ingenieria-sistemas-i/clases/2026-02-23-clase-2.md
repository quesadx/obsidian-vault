## Ingeniería de Software
Cada vez más las economías de los países desarrollados dependen de software. La ingeniería de software es la disciplina de la ingeniería que concierne teorías, métodos y herramientas para el desarrollo profesional de software. El gasto en esta disciplina representa un alto porcentaje del PIB de los países desarrollados. 
### Enfoque profesional 
Los ingenieros de software adoptan un enfoque sistemático para llevar a cabo un trabajo. Utilizan las herramientas y técnicas necesarias para resolver el problema planteado, considerando siempre restricciones de desarrollo y los recursos disponibles. Este enfoque metódico distingue la ingeniería del simple desarrollo de código, garantizando soluciones robustas.
### Historia y Evolución de la Ingeniería de Software
La ingeniería de software surgió como respuesta a la creciente complejidad y los desafíos inherentes al desarrollo.
- **1960's:** La "crisis del software" evidenció la falta de métodos formales. 
- **1968:**  Conferencia de la OTAN: nace el término "Ingeniería de Sistemas"
- **1970's-80's:** Programación estructurada y el modelo de cascada para mayor control.
- **1990's:** Consolidación de la programación orientada a objetos.
- **2000's:** Emergen metodologías ágiles para una entrega más rápida y flexible.
- **2010-Presente:** DevOPs, cloud computing e IA transforman el desarrollo.

## Qué es el software
### Productos de software
La mayor parte del gasto del software es en productos genéricos, pero hay más esfuerzo en el desarrollo de sistemas hechos a la medida. 
##### Productos genéricos
Productos que son producidos por una organización para ser vendidos al mercado abierto: Sistemas operativos, aplicaciones de oficina, Microsoft Office, Google Chrome. 
##### Productos personalizados
Sistemas que son desarrollados bajo pedido a un desarrollador específico: Sistemas empresariales, aplicaciones corporativas, sistema bancario interno, sistema ERP empresarial.
####  Características de los Productos de software
- **Mantenibles:** Debe ser posible que el software evolucione.
- **Confiabilidad:** No debe causar daños físicos o económicos en el caso de fallos del sistema.
- **Eficiencia:** No debe desperdiciar los recursos del sistema.
- **Utilización adecuada:** Debe contar con una interfaz adecuada. 
#### Importancia de las características del producto
Dependen del tipo de producto y del ambiente en que será utilizado. Los costos tienden a crecer exponencialmente si son requeridos altos niveles de alguna característica. 
### Ciclo de Vida del Software
1. **Planificación:** Define el alcance, objetivos y recursos del proyecto.
2. **Análisis de Requisitos:** Recopila y documenta las necesidades y expectativas del usuario.
3. **Diseño:** Crea la arquitectura del sistema y los componentes de software.
4. **Implementación:** Escribir el código fuente.
5. **Pruebas:** Verificar la funcionalidad y corrige para asegurar la calidad.
6. **Despliegue:** Pone el software a disposición de los usuarios finales. 
7. **Mantenimiento:** Actualiza, mejora y corrige el software después de su lanzamiento. 
8. **Retiro:** Fase final donde el software es descontinuado o reemplazado.

### Plataforma de cómputo
* **Sistemas embebidos:** Software integrado en dispositivos físicos con propósito específico.
* **Sistemas de tiempo real:** Software que debe responder dentro de límites temporales estrictos.
* **Sistemas de cómputo distribuido:** Software que opera en múltiples ordenadores conectados en red.
* **Sistema de cómputo obicuos:** Software presente en el entorno que interactúa de forma invisible.
* **Sistema de cómputo paralelo:** Ejecuta tareas en múltiples procesadores.
