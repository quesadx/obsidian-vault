## Bases de Datos

Las [[base-de-datos]] se usan principalmente en 2 tipos de procesos:
- **Online transaction processing:** Cuando una gran cantidad de usuarios emplean la bd para traer pequeñas cantidades de información, llevando a cabo pequeñas actualizaciones. 
- **Análisis de datos:** Procesar información para llegar a reglas o conclusiones, utilizadas para tomar decisiones de negocio. 

Las razones de la existencia de BD son:
- **Redundancia de datos e inconstencia:** El mismo dato podía aparecer repetido. 
- **Dificultad para obtener información:** Había que hacer las cosas manualmente. 
- **Datos aislados:** Datos en muchos archivos y formatos diversos.
- **Problemas de integridad:** Era difícil cumplir con diversas reglas. 
- **Problemas de atomicidad:** Las operaciones se realizaban en pasos, cuando deben ser indivisibles. 
	- **Problemas de acceso simultáneo:** Varias personas podían modificar el mismo dato al mismo tiempo. 
- **Problemas de seguridad:** No todos deben tener acceso a todo.

## Abstracción de Datos (ANSI/SPARC)

La arquitectura de tres niveles para describir una base de datos:

### Nivel Externo
Proyecta las capas de abajo de una forma abstracta, de manera que el usuario pueda interactuar con los datos.

### Nivel Conceptual
Describe qué datos están almacenados y sus relaciones.

### Nivel Interno
Describe cómo se almacenan físicamente los datos (archivos, índices). 

> [!Important] 
> Detrás de todas las bases de datos hay un sistema regular de archivos del sistema operativo.

## Esquema vs Instancia

### Esquema
Es el **diseño** de la base de datos. Define:
* Nombres de tablas
* Nombres de columnas
* Tipos de datos
* Constraints
* Relaciones

**Tipos:**
- **Esquema físico:** Describe el diseño a nivel físico
- **Esquema lógico:** Describe el diseño a nivel lógico

### Instancia
Los datos actuales en la base de datos en un momento dado.

> [!Importante] 
> Al final todas las bases de datos trabajan sobre un [[2026-02-19-sistema-operativo]], terminan implementando el manejo de archivo del SO. 
### Instancia
Una instancia es la base de datos como tal, almacenada en un momento y lugar específico.
## <mark style="background: #D2B3FFA6;">Proceso de diseño</mark>
Planificar para asegurar orden, eficiencia y seguridad. 
1. **Recolección de requisitos:** Qué necesitamos guardar?
2. **Modelado conceptual:** Diagramas alto nivel (ER)
3. **Modelado lógico:** Definición de claves y normalización
4. **Modelado físico:** Organización de archivos
5. **Implementación:** Código SQL en SABD

## <mark style="background: #D2B3FFA6;">SQL</mark>
Lenguaje declarativo para crear y manipular datos en modelos relacionales 
### DDL
Especifica el esquema de la bd. Define las estructuras de almacenamiento (tablas) y métodos de acceso.
- **Domain Constrains**: Posibles valores asociados con un atributo.
- **Integridad referencial:** Que se permita las relaciones entre atributos. 
- **Autorización:** Diferenciación de permisos de acceso entre usuarios. 
**Utiliza:** 
	**CREATE:** Crear objetos nuevos.
	**DROP:** Eliminar completamente.
	**ALTER:** Modificar. 
### DML
Expresa las queries. Maneja los datos (Insertar, Leer, Borrar). Hay 2 tipos de data-manipulation languaje:
- **Procedural DML:** Requiere que el usuario especifique que información requiere y como obtenerla. 
- **Declarative DML:** Requiere un usuario que especifique que información requiere, sin especificar como. 
	**Utiliza:** 
		**INSERT:** Insertar.
		**SELECT:** Consultar. 
		**UPDATE:** Actualizar.
		**DELETE:** Eliminar. 

> [!NOTE]
> Una query es una request solicitando información, la parte del DML que interactúa con esto es llamada query language. 

### DCL
Seguridad. Gestiona permisos y accesos.
	**GRANT:** Dar permisos.
	**REVOKE:** Quitarlos. 

## <mark style="background: #D2B3FFA6;">Instancia vs Base de Datos</mark>
La instancia es una imagen en ejecución en un momento dado. Es un mecanismo temporal para la manipulación de datos. SGA (memoria) + Procesos en Background, desaparece al apagar el servidor. 
La base de datos es el lugar físico de almacenamiento, datafiles + control files + redo log files. 
Un servidor oracle: Base de datos + instancia.
